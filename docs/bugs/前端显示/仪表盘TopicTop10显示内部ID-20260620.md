# 仪表盘 Topic Top10 显示内部 ID

## 根因分析

### 问题现象

仪表盘中 “Topic 调用 Top 10” 横向柱状图的 Y 轴显示类似 `t_xxx` 的 Topic 内部 ID。产品和运维人员无法直接判断它属于哪个命名空间、哪个 Topic，影响问题排查和业务观察。

### 涉及前端组件

- 前端页面：`frontend/src/views/Dashboard.vue`
- 图表渲染函数：`renderChart()`
- 具体展示位置：
  - Top10 模式通过 `ranking` 判断。
  - `labels = ranking ? [...stats.labels].reverse() : stats.labels`。
  - ECharts 横向柱状图的 `yAxis.data` 直接使用 `labels`。
  - `axisLabel` 直接展示 `labels`，没有转换成业务可读名称。

### 涉及后端接口

- 后端 Controller：`backend/src/main/java/io/litealert/admin/web/StatsController.java`
- 接口方法：`StatsController.ranking()`
- 接口：`GET /api/admin/stats/ranking`
- 当前逻辑：
  1. 从审计日志 `la_audit_log.attrs_json` 中读取 `topicId` 或 `apiKeyId`。
  2. 当 `dimension=TOPIC` 时，以 `topicId` 作为聚合 key。
  3. 返回 `labels` 时直接使用聚合 key：`top.stream().map(Map.Entry::getKey).toList()`。
  4. 因此前端收到的 `labels` 就是 Topic ID。

### 数据流转链路

1. Webhook / Notify 相关审计日志写入 `la_audit_log`，属性 JSON 中包含 `topicId`。
2. 仪表盘调用 `GET /api/admin/stats/ranking?dimension=TOPIC&limit=10`。
3. `StatsController.ranking()` 从审计日志读取 `topicId` 并按 ID 聚合 Top10。
4. 后端把 Topic ID 直接放入响应字段 `labels`。
5. 前端 `Dashboard.vue` 的 `renderChart()` 直接把 `stats.labels` 作为 ECharts Y 轴分类。
6. 最终 Y 轴显示内部 ID，而不是 `命名空间/Topic`。

## 解决方案

### 后端修复

- 修改 `backend/src/main/java/io/litealert/admin/web/StatsController.java`。
- 为 `StatsController` 注入 `TopicStore`。
- 在 `ranking()` 中，当维度为 `TOPIC` 时：
  - 将 Top10 的 Topic ID 转为 `namespaceName/name`。
  - 如果 Topic 已删除或查询不到，则回退显示原 Topic ID，避免图表空白。
- 保持 APIKey Top10 的 labels 逻辑不变。

### 前端兼容

- `frontend/src/views/Dashboard.vue` 继续使用后端返回的 `labels` 渲染。
- 适当加宽 Topic Top10 的左侧 grid / label 宽度，避免 `命名空间/Topic` 被过早截断。

# 命名空间表格域名访问出现横向滚动条

## 根因分析

- 涉及前端页面：`frontend/src/views/Namespaces.vue`。
- 涉及布局组件：`frontend/src/layouts/AppLayout.vue`。
- 具体表格配置：`Namespaces.vue` 中主表存在多个固定宽度列：ID 240、状态 110、Topic 90、已发布 90、禁用 80、草稿 80、创建时间 180、操作 220，再加展开列、名称列、描述列和页面左右 padding。
- 数据流转链路：用户访问 `/namespaces` → `Namespaces.vue` 调用 `/api/namespaces` 和 `/api/topics` → Element Plus `el-table` 根据列宽和容器宽度计算表格布局 → 当列宽总和超过可用内容宽度时出现横向滚动条。
- 当前现象“域名访问有横向滚动条，IP 访问没有”通常不是接口数据差异，而是浏览器对不同 origin 可能保存了不同的缩放比例、页面宽度、滚动条策略或缓存资源版本。域名和 IP 是两个不同站点，浏览器会分别保存 zoom；如果域名页面 zoom 比例更大，或者加载到旧 CSS，表格有效宽度就会变小，从而出现横向滚动条。

## 解决方案

1. 先确认域名和 IP 下浏览器缩放是否一致（建议都恢复 100%）。
2. 前端表格减少固定宽度总和：
   - ID 列可适当缩小或使用省略号。
   - 操作列可缩小或使用按钮组换行。
   - 统计列可合并为一个“Topic 概览”。
3. 给页面容器和表格增加更稳健的 overflow 控制，避免外层页面出现横向滚动条。
4. 如需彻底修复，建议在 `Namespaces.vue` 中对长 ID / 描述做 `show-overflow-tooltip`，减少硬编码列宽。

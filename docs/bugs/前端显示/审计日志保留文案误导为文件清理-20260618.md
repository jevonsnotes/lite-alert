# 审计日志保留文案误导为文件清理

## 根因分析

- 涉及前端页面文件：`frontend/src/views/AdminSystem.vue`
- 涉及后端实现：
  - `backend/src/main/java/io/litealert/common/audit/AuditLogger.java:log()`：通过 `insert into la_audit_log(ts, type, actor, trace_id, attrs_json)` 将审计事件写入数据库表。
  - `backend/src/main/java/io/litealert/common/audit/AuditJanitor.java:sweep()`：通过 `delete from la_audit_log where ts < ?` 清理超过保留期的数据库审计记录。
- 数据流转链路：管理员打开系统设置页 `/admin/system` → 前端 `AdminSystem.vue` 渲染“系统行为 / 审计日志保留”配置 → 页面模板中的固定说明文案展示给用户。
- 具体问题点：`AdminSystem.vue` 的“审计日志保留”说明仍沿用旧的文件审计日志描述 `audit/yyyy-MM-dd.log`，与当前数据库审计日志实现不一致，容易误导用户去服务器文件目录查找审计日志。

## 解决方案

- 修改 `frontend/src/views/AdminSystem.vue` 中“审计日志保留”的说明文案。
- 将文件清理描述替换为数据库记录清理描述：超过保留期的审计记录每天 03:17 从数据库中自动清理。
- 修改后运行前端类型检查，确认模板和 TypeScript 编译通过。

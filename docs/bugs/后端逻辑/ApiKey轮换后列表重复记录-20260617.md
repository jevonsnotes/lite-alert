# ApiKey 轮换后列表重复记录

## 根因分析

### 涉及前端组件/页面

- `frontend/src/views/ApiKeys.vue`
  - `rotate(k)` 调用 `POST /api/apikeys/{id}/rotate` 后执行 `load()` 重新加载列表。
  - 表格数据来自 `GET /api/apikeys`，前端没有主动复制记录。

### 涉及后端 Controller / Service / Store

- `backend/src/main/java/io/litealert/apikey/web/ApiKeyController.java`
  - `rotate(String id)`：调用 `ApiKeyService.rotate(id)` 并返回同一个 ApiKey 的视图。
- `backend/src/main/java/io/litealert/apikey/ApiKeyService.java`
  - `rotate(String id)`：在原 ApiKey 对象上更新 `prefix/keyHash/lastUsedAt/usageCount`，没有创建新 `id`。
- `backend/src/main/java/io/litealert/apikey/domain/ApiKeyStore.java`
  - `save(ApiKey k)` 调用 `index(k)`。
  - `index(ApiKey k)` 每次都会执行 `idByOwner.computeIfAbsent(...).add(k.getId())`。
  - 当同一个 ApiKey 被编辑、调用统计保存或轮换时，`idByOwner` 中会追加重复 id。
  - `findByOwner(ownerId)` 根据 `idByOwner` 列表逐个映射 `byId`，因此同一个 id 会被返回多次，前端列表表现为多出记录。

### 数据流转链路

1. 用户在 `ApiKeys.vue` 点击“轮换”。
2. 前端调用 `POST /api/apikeys/{id}/rotate`。
3. `ApiKeyController.rotate()` 调用 `ApiKeyService.rotate()`。
4. `ApiKeyService.rotate()` 原地更新同一条 ApiKey，并调用 `ApiKeyStore.save(k)`。
5. `ApiKeyStore.save()` 调用 `index(k)`，重复向 `idByOwner[ownerId]` 追加同一个 `id`。
6. 前端随后执行 `GET /api/apikeys`。
7. `ApiKeyService.listMine()` 调用 `ApiKeyStore.findByOwner()`，重复 id 映射成重复 ApiKey 对象。
8. `ApiKeys.vue` 表格渲染出重复记录。

## 解决方案

1. 修改 `ApiKeyStore.index(ApiKey k)`：
   - 写入 `idByOwner` 前先移除同 owner 下已有的相同 id，或只在不存在时添加。
   - `save(ApiKey k)` 中如旧对象存在，应确保 owner 索引不会重复。
   - `loadShard()` 加载历史文件时也应避免重复 id 写入索引。
2. 修改 `ApiKey` 实体，新增 `rotateCount` 字段，默认 0。
3. 修改 `ApiKeyService.rotate(String id)`：
   - 只更新同一条记录的 `prefix/keyHash`。
   - 清空 `lastUsedAt`，重置 `usageCount`。
   - `rotateCount + 1`。
   - 旧完整 Key 因 prefix/hash 被替换而自动失效。
4. 修改 `ApiKeyController.toView()`：
   - 返回 `rotateCount`，供前端展示。
5. 修改 `frontend/src/views/ApiKeys.vue`：
   - 增加“轮换次数”列。
6. 补充后端测试：
   - 同一个 ApiKey 多次 save/rotate 后，`findByOwner()` 只返回一条。
   - rotate 后 `rotateCount` 增加。

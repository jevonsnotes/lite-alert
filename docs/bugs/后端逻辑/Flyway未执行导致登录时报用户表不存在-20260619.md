# Flyway 未执行导致登录时报 la_user 不存在

## 根因分析

- 涉及后端登录链路：`backend/src/main/java/io/litealert/auth/AuthService.java:login()` 调用 `UserStore.findByUsername()`。
- 涉及后端 Store：`backend/src/main/java/io/litealert/auth/domain/UserStore.java:findByUsername()` 执行 `select * from la_user where lower(username) = lower(?)`。
- 涉及数据库迁移：`backend/src/main/resources/db/migration/*/V1__baseline.sql` 应创建 `la_user` 并插入默认 admin。
- 数据流转链路：应用启动 → 预期 Flyway 执行 V1/V2 migration → 用户登录 → 查询 `la_user`。实际启动日志中未出现 Flyway migration，说明当前运行环境没有执行 Flyway，数据库保持空库，登录时查询 `la_user` 报表不存在。
- 根本原因：Flyway 未在当前运行环境中执行。最可能原因是 IDE 启动配置未刷新 Maven 依赖，导致新增的 `flyway-core` 未进入运行时 classpath；也可能是运行环境禁用了 `spring.flyway.enabled` 或未使用最新构建产物。

## 解决方案

- 保持 Flyway 作为唯一正式建表和默认管理员初始化机制。
- 增加启动期 schema 校验：应用启动时检查核心表 `la_user` 和 `la_notify_delivery` 是否存在。
- 如果核心表不存在，启动阶段直接抛出清晰错误，提示 Flyway 未执行 / 数据库未初始化，而不是等到登录或 Worker 访问时报 SQL 异常。
- 使用测试验证空库在正常 Flyway 环境下可以创建核心表；若 Flyway 未执行，校验器能明确失败。

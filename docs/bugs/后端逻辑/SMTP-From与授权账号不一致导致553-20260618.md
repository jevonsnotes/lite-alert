# SMTP From 与授权账号不一致导致 553

## 根因分析

### 涉及前端组件/页面

- `frontend/src/views/AdminSystem.vue`
  - SMTP 配置表单允许分别填写 `username` 和 `fromAddress`。
  - 保存时直接提交 `fromAddress`，没有提示部分 SMTP 服务要求 From 与授权账号一致。

### 涉及后端 Service / Channel

- `backend/src/main/java/io/litealert/notify/mail/MailService.java`
  - `sendTest()` 中仅当 `fromAddress` 非空时设置 From。
  - 如果 `fromAddress` 与 SMTP 登录账号 `username` 不一致，部分 SMTP 服务会拒绝发送。
- `backend/src/main/java/io/litealert/notify/channel/EmailChannel.java`
  - `applyFrom()` 优先使用 `fromAddress`，否则使用 `username`。
  - 当配置了不一致的 `fromAddress` 时，同样可能触发 553。

### 数据流转链路

1. 管理员在系统设置中保存 SMTP 配置。
2. 配置中 `username` 与 `fromAddress` 不一致。
3. 邮件测试或通知发送时，JavaMail 设置邮件 From。
4. SMTP 服务端校验 From 必须等于授权账号。
5. 服务端返回：`553 Mail from must equal authorized user`。
6. 前端显示测试失败或通知派发失败。

## 解决方案

1. 后端保存 SMTP 配置时规范化 From：
   - 若 `username` 非空，则 `fromAddress` 默认使用 `username`。
   - 若用户填写的 `fromAddress` 与 `username` 不一致，为避免 553，自动改为 `username`。
2. `MailService.sendTest()` 设置 From 时复用统一逻辑：
   - 优先使用规范化后的 `fromAddress`。
   - 无 `fromAddress` 时使用 `username`。
3. 前端提示文案调整：
   - From 地址建议与 SMTP 用户名保持一致。

# 命名空间和 Topic 无法编辑名称

## 根因分析

### 问题现象

产品反馈命名空间和 Topic 创建后无法修改名称。该问题影响用户调整配置命名的体验，也影响复制后默认名称需要二次调整的场景。

### 根因 A：命名空间名称没有前端编辑入口，后端 PATCH 只支持描述

- 前端组件：`frontend/src/views/Namespaces.vue`
  - 命名空间列表仅通过 `el-table-column prop="name"` 展示名称。
  - `newNamespace()` / `submitNamespace()` 仅用于新建命名空间。
  - 操作列仅包含 `+ Topic`、禁用/恢复、删除，没有编辑名称入口。
- 后端 Controller：`backend/src/main/java/io/litealert/namespace/web/NamespaceController.java:update()`
  - `PATCH /api/namespaces/{id}` 接收 `UpdateRequest`。
  - `UpdateRequest` 仅包含 `description` 字段。
  - Controller 调用 `NamespaceService.updateDescription()`，不会处理 `name`。
- 后端 Service：`backend/src/main/java/io/litealert/namespace/NamespaceService.java:updateDescription()`
  - 只更新 `description` 和 `updatedAt`。
  - 名称格式校验和唯一性校验只存在于 `create()`。
- 数据流转链路：
  1. 前端通过 `GET /api/namespaces` 获取命名空间列表。
  2. 页面只渲染 `name`，没有名称输入框或编辑弹窗。
  3. 即使前端提交 `PATCH /api/namespaces/{id}` 携带 `name`，后端请求体也不会接收该字段。
  4. `NamespaceService.updateDescription()` 不更新名称，因此名称无法变更。

### 根因 B：Topic 名称没有已有对象编辑入口，后端 PATCH 请求体不包含 name

- 前端组件：`frontend/src/views/TopicDetail.vue`
  - 新建 Topic 页面使用 `draftCreate.name` 输入名称。
  - 已有 Topic 详情页标题展示 `topic.namespaceName / topic.name`。
  - “基础信息”Tab 只提供 ID 和描述，描述通过 `PATCH /topics/{id}` 单字段保存。
  - 没有已有 Topic 名称输入框。
- 后端 Controller：`backend/src/main/java/io/litealert/topic/web/TopicController.java:update()`
  - `PATCH /api/topics/{id}` 调用 `TopicService.update()`。
- 后端 Service：`backend/src/main/java/io/litealert/topic/TopicService.java:update()`
  - `UpdateRequest` 不包含 `name` 字段。
  - `update()` 只处理 `description`、`templates`、`notifyTemplate`、`transform`、`inboundFormat`、`auth`。
  - 名称格式校验和同命名空间唯一性校验只存在于 `create()`。
- 数据流转链路：
  1. 前端通过 `GET /api/topics/{id}` 获取 Topic 详情。
  2. 页面标题展示 `topic.name`，基础信息中没有名称编辑控件。
  3. 描述变更只提交 `{ description }`。
  4. 后端 `UpdateRequest` 不接收 `name`，Service 也不更新名称，因此名称无法变更。

### 设计约束

`docs/design/04-namespace-topic.md` 说明：

- 命名空间编辑通常只修改描述等非 URL 字段。
- Topic 的 `name` 影响 Webhook URL，DRAFT 可改，PUBLISHED 不可改。

因此该问题不是运行时异常，而是名称编辑能力缺失。修复时需要采用受控改名，避免破坏已接入调用方的 Webhook URL。

## 解决方案

### 命名空间受控改名

- 修改 `backend/src/main/java/io/litealert/namespace/web/NamespaceController.java`
  - `UpdateRequest` 增加 `name` 字段。
  - `update()` 调用新的 `NamespaceService.update()`。
- 修改 `backend/src/main/java/io/litealert/namespace/NamespaceService.java`
  - 新增或替换为 `update(String id, String name, String description)`。
  - 如果传入 `name` 且发生变化：
    - 使用 `NAME_RE` 校验格式。
    - 使用 `NamespaceStore.findByName()` 校验全局唯一，排除自身。
    - 更新命名空间名称。
    - 同步更新该命名空间下所有 Topic 的 `namespaceName`。
    - 记录审计日志。
- 修改 `frontend/src/views/Namespaces.vue`
  - 增加命名空间编辑入口。
  - 弹窗提示：命名空间名称会影响该空间下所有 Topic 的 Webhook URL。

### Topic 受控改名

- 修改 `backend/src/main/java/io/litealert/topic/TopicService.java`
  - `UpdateRequest` 增加 `name` 字段。
  - `update()` 在 Topic 为 DRAFT 时允许修改名称。
  - 如果 Topic 为 PUBLISHED 或 DISABLED 且请求修改名称，返回冲突错误。
  - 使用 `NAME_RE` 校验格式，并使用 `TopicStore.findByNamespaceAndName()` 校验同命名空间唯一，排除自身。
- 修改 `frontend/src/views/TopicDetail.vue`
  - 基础信息区增加 Topic 名称输入。
  - DRAFT 状态允许保存名称。
  - PUBLISHED / DISABLED 状态禁用名称输入，并提示“名称影响 Webhook URL，发布后不可修改”。

# 仪表盘 Topic Top10 搜索需求分析

**日期**：2026-06-20

## 需求概述

仪表盘当前的 “Topic 调用 Top 10” 只能展示调用量最高的 10 个 Topic。产品希望在 Top10 图表上增加搜索能力：

- 默认仍展示 Topic 调用 Top10。
- 用户可以搜索并选择任意 Topic，即使该 Topic 不在 Top10 内，也能查看它在当前时间窗口内的调用统计。
- 清空/删除搜索条件后，图表恢复默认 Top10。

通俗解释：Top10 是“排行榜”，但排在第 11 名、第 20 名甚至调用量很少的 Topic 也需要临时查看；搜索就是给排行榜增加一个“定点查看”能力。

## 技术可行性结论

可行。当前系统已有：

- 前端 `frontend/src/views/Dashboard.vue` 已加载全部 Topic 列表，可用于搜索选择。
- 后端 `GET /api/admin/stats/ranking` 已能按时间窗口聚合 Topic 调用数据。
- 只需扩展 ranking 接口支持 `topicId` / `apiKeyId` 参数：
  - 未传筛选 ID：返回 Top10。
  - 传入筛选 ID：返回该 Topic 或 ApiKey 的单条聚合统计，即使它不在 Top10 内。

## 推荐方案及理由

### 1. 后端接口扩展

扩展接口：

```http
GET /api/admin/stats/ranking?dimension=TOPIC&limit=10&topicId=<topicId>
```

规则：

- `dimension=TOPIC` 且 `topicId` 为空：保持现有 Top10 行为。
- `dimension=TOPIC` 且 `topicId` 不为空：只统计该 Topic，并返回一条记录。
- 该记录的 `labels` 仍使用已修复的 `namespaceName/topicName` 展示。
- 如果 Topic 没有调用数据，也返回该 Topic 的 0 值统计，避免用户搜索后看到空图误以为功能坏了。

保留 APIKey Top10 当前行为，本需求只处理 Topic Top10。

### 2. 前端交互设计

修改 `frontend/src/views/Dashboard.vue`：

- 当 `topicMode === 'RANKING'` 时，在 Top10 / 趋势切换右侧显示一个可搜索、可清空的 Topic 下拉框。
- 下拉框内容使用已有 `topics` 列表，显示 `namespaceName/topicName`。
- 用户选中 Topic：
  - 保存到 `rankingTopicId`。
  - 调用 `/admin/stats/ranking` 时附带 `topicId`。
  - 图表变为单条横向柱状图，展示该 Topic 当前时间窗口内的受理/送达/失败数。
  - 标题可显示 `Topic 调用：命名空间/Topic`，帮助用户知道当前不是 Top10。
- 用户清空下拉框：
  - 清空 `rankingTopicId`。
  - 重新请求默认 Top10。
  - 标题恢复 `Topic 调用 Top 10`。

## 风险点和注意事项

1. **0 调用 Topic 的展示**
   - 搜索到一个从未被调用过的 Topic 时，推荐显示一条 0 值柱状图，而不是空图。
   - 这样用户能明确知道“该 Topic 在当前窗口无调用”。

2. **性能风险较低**
   - 当前统计来自审计日志窗口聚合，新增 `topicId` 过滤只是在已有循环中增加匹配条件。
   - Topic 下拉框复用已加载 Topic 列表，不新增额外列表接口。

3. **交互语义**
   - 搜索选中时，图表不再是 Top10，而是“指定 Topic 统计”。
   - 需要通过标题或提示让用户知道当前处于搜索查看状态。

## 预估工作量

- 后端接口扩展与测试：0.5 天。
- 前端下拉搜索、清空恢复 Top10 与构建验证：0.5 天。

整体预估：约 0.5–1 个工作日。

## 产品确认结果

1. 搜索选中 Topic 后，按推荐方案显示“单个 Topic 的横向柱状统计”。
2. 搜索到当前窗口无调用的 Topic / ApiKey 时，显示 0 值柱状图。
3. 搜索范围扩展为 Topic Top10 和 ApiKey Top10 都支持搜索；清空搜索后分别恢复默认 Top10。
